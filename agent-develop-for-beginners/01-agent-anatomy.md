# 第一章：解剖一个 AI Agent

想象一下，你要在公司里招聘一名新员工来专门负责“竞品调研”。
在这个过程中，这名新员工需要具备什么？

1. **职位描述 (Profile)**：他要知道自己的身份和工作目标（你是调研员，不是销售）。
2. **大脑 (Brain)**：他需要有逻辑思考能力，能理解任务、拆解步骤。
3. **手脚 (Tools)**：他需要会使用搜索引擎、Excel、或者公司内部的数据终端。
4. **记忆 (Memory)**：他得记住昨天做过什么，查过哪些资料，公司以前的竞品报告放在哪。

其实，构建一个 AI Agent（智能体）的逻辑，与雇佣一名员工出奇地相似。在学术界和工业界，目前普遍公认的 Agent 核心架构，正是由以上四个部分组成。

## 1. 角色设定（Profile / Persona）

如果你直接问一个裸大模型（LLM）问题，它可能会以一个“乐于助人的 AI 助手”来回答。但在 Agent 开发中，我们首先需要给它注入一个“灵魂”——角色设定。

角色设定通常通过核心的 `System Prompt`（系统提示词）来奠定。它决定了 Agent 的**边界、语气、以及它的主要目标**。

```python
agent_profile = {
    "role": "资深金融数据分析师",
    "goal": "通过分析股票数据，为客户提供中立、客观、有数据支撑的投资报告。",
    "backstory": "你在华尔街有10年工作经验，极其严谨，从不做出毫无依据的预测。"
}
```

## 2. 大脑（Brain / LLM）

大模型是 Agent 的灵魂，但它在这里扮演的角色不仅是一个“回答问题的字典”，而是一个**中央处理器（CPU）**。

当 Agent 收到用户的任务时，Brain 需要进行：
- **感知与理解**：用户真正的意图是什么？
- **推理与规划（Reasoning & Planning）**：为了完成这个任务，我需要分几步走？如果第一步失败了怎么办？

我们在第二章会深入探讨 Brain 是如何利用诸如 `ReAct`（推理与行动）和思维链等模式来解决复杂问题的。

## 3. 双手与工具（Tools & Actions）

大模型本身的知识是**静态的**（停留在它的训练截止日期），并且**它只能输出文本**。

为了让 Agent 能够与物理世界或外部系统交互，我们需要给它装备工具（Tools）。
工具可以是任何一段有明确输入输出的代码，例如：
- 搜索网页的 API。
- 执行本地 Python 脚本的环境。
- 读取数据库表结构的函数。
- 操作浏览器进行网页点击的自动化脚本（例如 OWL 框架中的 Playwright 工具）。

Agent 的 Brain 会决定**何时**去调用**哪个**工具，并使用工具返回的真实结果来继续下一步推理。

## 4. 记忆（Memory）

人类在处理复杂工作时，有随时查阅笔记的习惯。Agent 同样需要强大的记忆系统。

Agent 的记忆一般分为两大类：
- **短期记忆（Short-term Memory）**：上下文记忆。就像我们在连续对话中，它记得你前一句话说过的约束条件。通常受限于大模型的上下文窗口（Context Window），如 128K 或 200K tokens。
- **长期记忆（Long-term Memory）**：外部存储。把历史对话、文档资料向量化（Embedding）后存入向量数据库（Vector Database）。当 Agent 需要时，通过 RAG（检索增强生成）技术，瞬间把过去多年的“工作经验”回调到大脑里。

---

## 小结

可以说：**Agent = 角色设定 + LLM大脑 + 记忆系统 + 外部工具**。

这就是 AI 从“能言善辩的聊天机器人”进化为“独立完成任务的数字员工”的核心秘方。
